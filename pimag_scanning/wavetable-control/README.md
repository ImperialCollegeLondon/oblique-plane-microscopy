# Wavetable control

This folder is a placeholder for C-413 wave generator / wavetable control notes.

The C-413 can generate motion internally from wave tables. This can be useful when the desired scan should be generated by the controller rather than by NIS-Elements analog stepping or by a single `MOV` command with position-distance triggers.

## Basic command pattern

```text
WTR <WaveGenID> <WaveTableRate> <InterpolationType>
WAV <WaveTableID> <AppendMode> <WaveType> <WaveTypeParameters>
WSL <WaveGenID> <WaveTableID>
WGC <WaveGenID> <NumberOfCycles>
WOS <WaveGenID> <Offset>
WGO <WaveGenID> <StartMode>
```

A simple linear test from the legacy notes is:

```text
WTR 1 2 0
WAV 1 X LIN 2000 0.2 0 2000 0 200
WSL 1 1
WGC 1 10
WOS 1 0.6846
WGO 1 1
```

## Timing

The output duration is:

```text
Output duration = servo_cycle_time * WTR_value * number_of_wave_points
```

The C-413 manual example uses a default servo cycle time of `200 µs`. A `WTR` value of `1` outputs one wave-table point per servo cycle. Increasing `WTR` increases the duration of the waveform.

## Important interactions

- When the wave generator is active for an axis, normal motion commands such as `MOV` are not allowed for that axis.
- If analog input control is enabled for the same axis, the analog input can take over target/control value generation; the wave generator may remain active but its output will no longer be used for the axis.
- Keep wavetable testing separate from the NIS analog-control setup until the ownership of the axis target is completely clear.
